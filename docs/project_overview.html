用户态 TCP/IP · TUN · C++17

TinyTCP-Over-TUN

基于 Linux TUN 设备的用户态 TCP/IP 栈，实现 ARP/IP 路由、TCP Reno、重组器优化与 HTTP Demo。

内置 web_server 现已支持多次刷新：每个请求都会创建独立的 Minnow TCP 套接字。
核心亮点
  • 用户态替代：拦截 TUN 数据报，重实现 TCP 三次握手/四次挥手。
  • TCP Reno：Slow Start、CA、Fast Retransmit/Recovery。
  • 性能：环形缓冲重组器 + 二叉前缀树路由，Benchmark 提升显著。
  • HTTP Demo：web_server 每请求新建连接，多次刷新稳定返回。
  • 一键环境：env_manage.sh 启停 TUN + DNAT，VPS 快速暴露服务。

快速启动 (Docker)

进入开发容器并构建：

docker-compose run --rm dev /bin/bash
cmake -S . -B build
cmake --build build -j

测试套件：

cmake --build build --target check6
cmake --build build --target speed

VPS 一键运行 web_server

env_manage.sh 封装 TUN 与 DNAT，写好 ETH_DEV/PORT 后：

sudo ./env_manage.sh start
./build/apps/web_server -l 169.254.144.9 8080
# 浏览器访问 http://<公网IP>:8080，刷新多次
sudo ./env_manage.sh stop

HTTP 客户端示例

webget 通过用户态 TCP 访问外部站点：

sudo ./scripts/tun.sh start 144
./build/apps/webget stanford.edu /class/cs144/

配合 tcpdump -i tun144 可观察用户态包收发。

架构速览

链路层 · Ethernet/ARP 处理，TUN 设备注入/读取。
网络层 · IPv4 路由，二叉前缀树 LPM，加工校验和/TTL。
传输层 · TCP 状态机、重传定时器、拥塞控制。
会话 · TCPMinnowSocket 双线程（事件循环 + 前台 API）。
应用 · webget、web_server Demo。

关键特性

  • 环形缓冲区重组乱序数据，消除头部 erase 的 O(N) 开销。
  • 超时重传 & 自适应 RTT 估计，支持部分 ACK 处理。
  • 单连接 Minnow 套接字：web_server 为每个请求创建全新实例。
  • iptables DNAT 一键转发：外网 {ETH_DEV}:{PORT} → 内部 169.254.TUN.9。
